🎯 가장 간단한 해결 방법 (30초 소요)
========================================================================

Railway Dashboard에서 Environment Variable 추가로 강제 재배포:

STEP 1: Railway Dashboard 열기
--------------------------------
https://railway.app/project/53c2f700-32ca-491f-b525-8552114b6fd6

STEP 2: Service 선택
--------------------------------
"warm-lead-generation" 클릭

STEP 3: Variables 탭
--------------------------------
상단 메뉴에서 "Variables" 클릭

STEP 4: 새 Variable 추가
--------------------------------
"New Variable" 버튼 클릭

입력:
- Variable Name: FORCE_REBUILD_V2
- Variable Value: true

"Add" 버튼 클릭

STEP 5: 자동 배포 시작
--------------------------------
Variable이 추가되면 자동으로 새 배포가 시작됩니다!
"Deployments" 탭에서 진행상황 확인 가능

STEP 6: 1-2분 대기
--------------------------------
빌드 및 배포 완료까지 대기

STEP 7: 검증
--------------------------------
배포 완료 후 실행:

cd C:\Users\seans\nerdx-apec-mvp\warm-lead-generation
verify_deployment.bat

또는:

curl -X POST "https://warm-lead-generation-production.up.railway.app/api/v1/lead-scoring/calculate" ^
  -H "Content-Type: application/json" ^
  -d "{...premium lead data...}"

예상 결과:
{
  "nbrs_score": 99.25,   ← 99여야 함 (15.17 아님!)
  "tier": "tier1",       ← tier1이어야 함 (tier4 아님!)
  ...
}

========================================================================
왜 이 방법이 작동하는가?
========================================================================

Railway는 Environment Variable이 변경되면:
1. 캐시를 무시하고
2. GitHub에서 최신 코드를 다시 pull하고
3. 완전히 새로운 빌드를 실행합니다

이 방법이 가장 확실하게 최신 코드를 배포합니다!
